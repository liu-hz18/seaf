seafquant 层还没有实现，目前的规划分为：数据加载节点、多因子计算节点（多个）、模型训练和生成信号节点、信号IC分析节点、分层回测选股策略绩效分析节点。这样来组成一个完整的量化分析框架。

3. 多因子计算节点：根据你对量化因子专业的理解和深度梳理，构造128个时序因子和截面因子，将因子分为若干类之后，每一大类在同一个节点中计算。这样设计主要是为了复用不同因子之间相似或相同的计算结构，并充分利用节点之间的多进程提高计算效率。因子在输出到queue之前都需要进行截面的标准化，防止量纲不一致导致下游训练和分析出现问题；你还需要注意因子部分的平稳性，以提高下游模型训练的成功率; 根据我们的流式框架（框架本身就是时序滚动传输数据的），因为节点每次只输出 1 天（最新的1个交易日）的计算结果，所以在因子计算节点中，你可以规避很多在常规计算中 rolling 的计算操作，而直接计算节点期望输出的“当天”的计算结果。你可能需要改造 frame3d 类来实现 时序计算的api和截面计算的api，或者使用其他更兼顾效率和扩展性的方式；

5. 信号IC分析节点：对于模型节点给出的预测信号（每天都有一个截面向量），我们计算其与 未来20日截面超额收益率向量 的相关系数，也就是当天的 截面IC，来度量信号当天的质量。每个交易日都会有一个IC数据，我们将每日的ic数据记录在 mlflow 中。同时，还需要记录 cumsum IC, 也就是每日的IC的前缀累计值。（这你可能需要改造 qpipe\node.py 中的 multiinputnode 类，来补充一个 context 参数，以实现有状态的node，来支撑前缀和的计算；
